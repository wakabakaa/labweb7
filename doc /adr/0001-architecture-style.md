# 1. Вибір архітектурного стилю: Модульний моноліт

Date: 2025-12-09
Status: Accepted

## Context
Ми розробляємо систему бронювання для ресторану/готелю. Команда невелика, домен ще досліджується. Потрібно швидко стартувати, але зберегти можливість масштабування.

## Decision
Ми вибираємо **Модульний Моноліт**.

## Consequences
Pros:
* Легкий деплой (один артефакт).
* Швидкий рефакторинг між модулями.
* Простіше тестування (немає мережевих затримок).

Cons:
* Можливе сильне зв'язування, якщо не слідкувати за межами модулів.
